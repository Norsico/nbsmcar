# 定时器分配

当前按 `Seekfree_STC32G144K` 这套工程来记，后面改动先同步这里，免得忘。

## 1. 当前已定

| 模块 | 资源 | 周期 / 触发 | 优先级 | 说明 |
| --- | --- | --- | --- | --- |
| 摄像头采集 | `INT1` | `VSYNC` | `3` | 逐飞库里已经设成最高，别动。 |
| 摄像头搬运 | `DMA_LCM` | 一帧完成 | `3` | 跟图像采集同级，尽量别压低。 |

| UI按键扫描 | `TIM5_PIT` | `10ms` | `0` | 当前 `ui.c` 已经在用，只置按键标志。 |
| UI屏幕刷新 | `TIM6_PIT` | `50ms` | `0` | 当前 `ui.c` 已经在用，只置刷新标志。 |

| 模块 | 资源 | 周期 / 触发 | 优先级 | 说明 |

## 3. 定时器注意事项

1. `TIM2` 不要用。
原因：逐飞库里已经把它当串口波特率发生器用了，`PIT` 和 `ENCODER` 都不要再占它。

2. 一个定时器只能干一种事。
原因：逐飞库内部会用 `timer_funciton_check()` 卡冲突。

3. 控制环分开，不要揉一个总定时器里。
当前方向：电机一个定时器，舵机一个定时器，UI按键一个定时器，UI屏幕一个定时器，WiFi一个慢定时器。

4. 图像不要塞进控制定时器。
图像继续走相机中断 + DMA + 主循环按帧处理。

5. `TIM3_PIT` 先按默认优先级用。
原因：当前库里没有直接给 `TIM3_PIT` 单独设优先级的接口枚举，不要自己脑补成 `1`。

6. `ui.c` 里的两个定时器只置标志。
按键扫描和屏幕刷新都放主循环里做，不要在中断里读屏、刷屏、改页面。

## 4. 优先级原则

1. 图像采集链最高。
2. 电机、舵机闭环次高。
3. 陀螺仪和慢任务再往后。
4. 屏幕、蜂鸣器、激光笔最低。

## 5. 备注

- 这版开始，除了图像处理和确实必要的地方，默认优先用 `<=16` 位数据。
- `motor.c` 里编码器初始化和电机初始化分开写，但仍收在一个文件里。
